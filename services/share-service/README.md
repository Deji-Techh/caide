# CAIDE Share Service

Private snapshot-sharing API and HTTPS landing page for `.caidepkg` files.

## Automated tests

```sh
npm install && npm run build && npm test
```

The tests use mocks and do not require PostgreSQL, MinIO, Docker, or any cloud services.

## Production

Provision PostgreSQL and a private S3/R2 bucket, replace every development
credential, set `SHARE_PUBLIC_BASE_URL=https://share.caide.app`, provide real
CAIDE installer URLs, apply the migration, and deploy behind HTTPS. Clients
receive only short-lived signed upload and download URLs.

## Before production traffic

Use a shared rate-limit store when running more than one API replica, configure an
object lifecycle rule as a second layer of expired-file cleanup, and commit the
`package-lock.json` generated by `npm install` so production images can switch to
`npm ci`.
